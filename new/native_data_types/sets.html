Sets are a new datatype added in Python 2.7. They are a cross between lists and dictionaries. Sets guarantee only unique values will be present. A set in Python is like an instance of Set in Java

3.8.1 Defining Sets

Example 3.29. Defining a Set

>>> d = { "a", "b", "b", "mpilgrim", "z", "example" } 
>>> d
set(['a', 'mpilgrim', 'z', 'b', 'example']) 
>>> d[0] 
>>> empty_dictionary = {} 
>>> empty_dictionary
{ }
>>> empty_set = set() 
>>> empty_set
set( [ ] )

To create a set, you put comma separated values in curly braces. The set will automatically remove any duplicates.

Sets are an unordered data structure. It will likely come out in a different order than you put the items in.

Sets cannot be indexed like lists, because they are unordered.

Empty curly braces create a dictionary, even though sets use curly braces.

You need to call set() to create an empty set.

Example 3.30. Modifying a Set

>>> d
set( ['a', 'mpilgrim', 'z', 'b', 'example'] )
>>> d.add('c') 
>>> d
set( ['a', 'c', 'b', 'mpilgrim', 'z', 'example'] )
>>> d.add(‘b’) 
>>> d
set( ['a', 'c', 'b', 'mpilgrim', 'z', 'example'] )
>>> d.add('A') 
set( ['a', 'A', 'c', 'b', 'mpilgrim', 'z', 'example'] )
>>> d.add(0) 
set( ['a', 'A', 'c', 'b', 0, 'mpilgrim', 'z', 'example'] )

The add method is used to add an item to the Set.

Attempting to add an item to the Set that already exists does not raise an exception. It does not change the Set, either.

Like dictionaries, items in Sets are case-sensitive. ‘A’ is considered unique from ‘a’.

Sets can contain multiple types, such as strings and integers in the same Set. Adding ‘0’ would work, because ‘0’ is not equal to 0. The first is a string, and the second is an integer.

Trying to remove an item that is not in the Set does raise a KeyError exception.

3.8.2 Deleting Items From Sets

Example 3.31. Deleting Items from a Set

>>> d
set( ['a', 'A', 'c', 'b', 0, 'mpilgrim', 'z', 'example'] )
>>> d.remove(‘a’) 
set( ['A', 'c', 'b', 0, 'mpilgrim', 'z', 'example'] )
>>> d.remove('f') 
Traceback (most recent call last):
  File "", line 1, in 
KeyError: 'f'
>>> d.clear() 
set( [ ] )

The only way to remove an item from a set is with the remove method.

You cannot use del to remove an item like a dictionary.

To clear a set, simply use the clear method.

One of the common uses for sets is comparing two or more sets. An example is finding the union or intersection of two sets.

X.X.X Set Operations 

>>> a = {'a', 'b', 'x'}

>>> b = {'x', 'y', 'z'}

>>> a
set( ['a', 'x', 'b'] )

>>> b
set( ['y', 'x', 'z'] )

>>> a.union(b) 
set( ['a', 'b', 'y', 'x', 'z'] )

>>> a.intersection(b) 
set( ['x'] )

>>> a.difference(b) 
set(['a', 'b'])

>>> b.difference(a)
set( ['y', 'z'] )

You can use union() to find the set of all the elements in both sets, ignoring duplicates like any set. It can also be written a | b. b.union(a) or b | a would produce the same result.

intersection() finds all the elements that occur in both sets. This can also be written a & b, and order is not important.

Calling difference from a on b will find all the elements that occur in a but do not occur in b. Difference can also be (predictably) written a - b. Unlike union and intersection, order matters.

There are other functions that you can use with sets, such as updating a set from another set. These functions are discussed in the documentation.
